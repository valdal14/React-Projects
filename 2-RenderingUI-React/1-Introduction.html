Rendering UI Intro

Instead of using String templates, React uses JS Objects to build the UI React Elements is just light JS Objects not actual DOM nodes We can create our own custom Elements called Components

Creating Elements

React supported HTML Attributes that can be passed as properties inside an element:

React.createElement( /* type */, /* props */, /* content */ );

  • type Either a string i.e 'div' or a React Component (JSX syntax soon)
  • props Either null or an object
  • content â€“ null A string, a React Element, or a React Component
  • Anything that you pass here will be the content of the rendered element. This can include plain text, JavaScript code, other React elements, etc.


// EXAMPLE: React.createElement( /* type */, /* props */, /* content */ );

const element = React.createElement('h1', {
  // properties
  className: 'welcome-message'
},


Rendering Elements onto the DOM

React introduced the concept of VIRTUAL DOM

VIRTUAL DOM: Objects that describe real DOM nodes

// Render the react element we created above

ReactDOM.render(
  element,
  document.querySelector('#root')
)


Nesting

// Nesting elements

const element = React.createElement('h1', {
  // properties
  className: 'welcome-message'
},
  // NESTING
  React.createElement('span', null, 'Hello React')
);


// Nesting example

const element = React.createElement('ol', {
  // properties
  className: 'welcome-message'
},
  // NESTING
  React.createElement('li', null, 'Milk'),
  React.createElement('li', null, 'Coffee'),
  React.createElement('li', null, 'Bread')
);


// Better Nesting example

const food = [
  { id: '0', product: 'Milk' },
  { id: '1', product: 'Coffee' },
  { id: '2', product: 'Bread' }
]

const element = React.createElement('ol', {
  // properties
  className: 'welcome-message'
},
// NESTING
  food.map((product) => (
   React.createElement('li', {
      key: product.id
    },
    product.product)
  ))
);


// Nesting notes......
"Hello world!" will be wrapped in a 'div' when this React element renders as HTML. While we can indeed nest React elements, remember the overall call just returns a single element.

const element = React.createElement('div', null, React.createElement('strong', null, 'Hello world!') );

// JSX notes......
When writing JSX, keep in mind that it must only return a single element. This element may have any number of descendants, but there must be a single root element wrapping your overall JSX

const jsxElement = {food.map(product => { product.product } )}


Intro to Components

So far we've seen how .createElement() and JSX can help us produce some HTML. Typically, though, we'll use one of React's key features, Components, to construct our UI.

Components refer to reusable pieces of code ultimately responsible for returning HTML to be rendered onto the page. More often than not, you'll see React components written with JSX.

Since React's main focus is to streamline building our app's UI, there is only one method that is absolutely required in any React component class: render()

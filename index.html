Stateless vs Stateful Components


  1. Does not matter if you are using Functional or Class Components you can always differenciate them as Stateless and Stateful
  2. Stateful is a Functional or Class component that manages states
  3. Stateless is a component (like our Person.js Component) that does not have an internal state
  4. Stateless Components get external data and output them without managing states
  5. The best-practice is to create as many as Stateless Components as possible in our React app
